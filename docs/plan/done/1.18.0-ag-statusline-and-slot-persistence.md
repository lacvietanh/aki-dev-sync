# Implementation Plan: v1.18.0 - AGY Statusline Integration, Slot Account Persistence & Header UX Refinements

**Version Target:** `1.18.0` (bare semver in `package.json` & `Cargo.toml`)  
**Date:** 2026-07-22  
**Status:** Completed  
**Research & Architecture:** [`docs/research/statusline-multi-cli-account-isolation.md`](../../research/statusline-multi-cli-account-isolation.md)

---

## 1. Objectives

1. **Dual Statusline Deployment (Claude Code & AGY CLI via OOP Adapter Pattern)**:
   - Expand `src-tauri/src/statusline.rs` using an **OOP Adapter Pattern** (`StatuslineTarget` trait) so that clicking **Apply** in the Statusline Customizer deploys statuslines to both:
     - `~/.claude/statusline-command.sh` (Claude Code + patch `settings.json`)
     - `~/.gemini/antigravity-cli/statusline.sh` (AGY CLI, standalone executable)
   - Add `"account"` (Account Email) to the Statusline Field Catalog, enabling users to toggle and reorder session account visibility across terminal sessions.
   - For architecture, process isolation proof, and adapter design details, see [`docs/research/statusline-multi-cli-account-isolation.md`](../../research/statusline-multi-cli-account-isolation.md).

2. **Persistent Selected View & Account per Usage Slot**:
   - Save and restore each usage slot's selected account (`slotViewingEmail`) in `localStorage` under `aki-usage-slot-${props.slotId}-viewing-account`.
   - **Defensive Error Guard**: Gracefully fallback to default active account (`null`) if the persisted account is deleted or offline cache is missing.

3. **AppHeader Dropdown Menu Refinements**:
   - Clarify Statusline Customizer dropdown items with icons/labels for Claude Code & AGY CLI.
   - Add visual separator `<div class="icon-dropdown-separator"></div>` between **Usage row** options and window resize buttons.
