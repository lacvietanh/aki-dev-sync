# Plan: AG Usage & Account List Enhancements (v1.16.1)

Status: Completed (1.16.1)
Target Version: 1.16.1 (Patch Release)
Date: 2026-07-22

## Overview

This release introduces targeted refactorings and UX/SRP improvements for the Antigravity (AG) usage monitoring component and multi-account dropdown list.

---

## Key Requirements & Architectural Scope

### 1. Single Responsibility Principle (SRP) for Indicators
- **Type Distinction (CLI vs IDE):** 
  - Expressed via an icon at the very **beginning of each account row** in the dropdown list (`<i class="fa-solid fa-terminal"></i>` for AGY CLI in purple, or `<img src="/antigravity-icon.png">` for AG IDE).
  - Compact sizing (`12px × 12px`) matching the default row line-height.
- **Live Status Indicator:**
  - Expressed via a single **"Live Green" dot (`#22c55e`)** on the right side of the row.
  - Replaces the legacy cyan/magenta color scheme for live dots (which conflated source type with live status).
- **Strict Live Detection Flow:**
  - An account record is ONLY marked as live if:
    1. It belongs to the current active email/account key from the live fetch, **AND**
    2. Its `fetchedAt` timestamp is recent (`nowSec - fetchedAt <= 600s` / 10 minutes), preventing historical/stale records (e.g., 337h old) from erroneously rendering live indicators.

### 2. Cache Eviction (10-Day Account Pruning)
- **Rule:** Accounts with `fetchedAt` older than 10 days (`> 864,000` seconds) are automatically evicted from local cache (`AG_CACHE_KEY`).
- **Placement:** Embedded cleanly in `loadAgStore()` and `listAgAccounts()` in `src/composables/useAgentUsage.js`, ensuring outdated accounts are purged at store load time without requiring manual user clearing.

### 3. Antigravity Header Title Renaming (Wide Mode)
- **Wide Mode (`window.innerWidth > 700px`):**
  - Displays **`agy`** when active source is `cli`.
  - Displays **`AG IDE`** when active source is `ide`.
- **Narrow Mode (`window.innerWidth <= 700px`):**
  - Remains hidden (`u-narrow-hide`), keeping only the icon visible for extreme-narrow layout efficiency.

### 4. Smart Popup Dropdown Placement Classes (4-Corner CSS Pattern)
- Implementation of clean, reusable CSS pattern classes for dropdown placement relative to trigger anchors:
  - `popup-pos-tl`: Top-Left anchor → Expands Downward-Right (`top: calc(100% + 4px); left: 0; origin: top left`).
  - `popup-pos-tr`: Top-Right anchor → Expands Downward-Left (`top: calc(100% + 4px); right: 0; origin: top right`).
  - `popup-pos-bl`: Bottom-Left anchor → Expands Upward-Right (`bottom: calc(100% + 4px); left: 0; origin: bottom left`).
  - `popup-pos-br`: Bottom-Right anchor → Expands Upward-Left (`bottom: calc(100% + 4px); right: 0; origin: bottom right`).

### 5. Stable Masked Email Display (Fixed Width Blur)
- **When Masked (`showEmail === false`):**
  - Always exposes the **first 4 characters** of the email (e.g. `lacv...`).
  - Followed by a blurred element forced to a **fixed width (`3rem`)**.
  - Prevents horizontal layout shifting during toggle, maintaining a smooth inline flow:
    `[Type Icon] -> [4 Chars] -> [Fixed Width Blur (3rem)] -> [Live Green Dot] -> [Relative Time]`.
