# Plan: Antigravity Multi-Surface Support - IDE, AGY CLI & Desktop App (v1.17.0)

Status: Planned / In Progress
Target Version: 1.17.0 (Minor Release)
Date: 2026-07-22

## Overview

This release introduces comprehensive multi-surface discovery and UI/UX unification for the **Google Antigravity ecosystem**, fully supporting all 3 execution surfaces:
1. **Antigravity IDE** (VS Code extension / IDE)
2. **AGY CLI** (Terminal command-line agent)
3. **Antigravity 2.0 Desktop App** (Standalone Electron App - "Bản Trắng")

---

## Architectural Scope & Core Requirements

### 1. Dual-Domain Auth Model (System Architecture)
Empirical analysis confirms the system operates on **2 Core Auth Domains**:
- **Domain 1 (IDE Storage):** `~/Library/Application Support/antigravity-ide/User/globalStorage/state.vscdb` (SQLite DB used exclusively by Antigravity IDE).
- **Domain 2 (Gemini Core Shared Storage):** `~/.gemini/` (Shared JSON credentials & OAuth tokens used simultaneously by **AGY CLI** and **Antigravity Desktop App**).

### 2. Multi-Surface Process Detection (`scripts/get-antigravity-usage.js`)
- **Binary Matching Update:** Add `'language_server'` to `BINARY_NAMES` to catch `/Applications/Antigravity.app/Contents/Resources/bin/language_server`.
- **Fingerprint-Based Classification (`sourceType`):**
  - Path containing `Antigravity IDE.app` or binary `language_server_macos_*` $\rightarrow$ `sourceType: "ide"`
  - Path containing `Antigravity.app` $\rightarrow$ `sourceType: "desktop"`
  - Command line running `agy` $\rightarrow$ `sourceType: "cli"`
- **Smart Deduplication:**
  - When both `AGY CLI` and `Antigravity Desktop App` are running on the same machine under the same account (`~/.gemini/`), deduplicate into a single consolidated snapshot (`sourceType: "desktop_cli"`) to avoid rendering duplicate identical cards.

### 3. Visual UI & Account Dropdown Branding (`src/components/AgentUsage.vue`)
- **Visual Assets:**
  - `/public/antigravity-app-icon.png` (Clean 64x64 PNG extracted from `/Applications/Antigravity.app/Contents/Resources/icon.icns`).
  - `/public/antigravity-icon.png` (Gradient IDE icon).
  - `<i class="fa-solid fa-terminal"></i>` (Terminal CLI icon).
- **Header Title Display Labels (Locked Spec):**
  - **`AG`**: For Antigravity Desktop App (`sourceType: "desktop"` - Bản trắng).
  - **`AG IDE`**: For Antigravity IDE (`sourceType: "ide"` - Bản IDE).
  - **`AGY`**: For AGY CLI (`sourceType: "cli"` - Terminal CLI).
  - **`AG / AGY`**: For consolidated Desktop App + CLI session (`sourceType: "desktop_cli"`).
- **Header & Dropdown Icon Rendering:**
  - `desktop` $\rightarrow$ White Antigravity Desktop App icon (`/antigravity-app-icon.png`).
  - `ide` $\rightarrow$ Gradient Antigravity IDE icon (`/antigravity-icon.png`).
  - `cli` $\rightarrow$ Terminal icon (`fa-terminal`).
  - `desktop_cli` $\rightarrow$ Combined White App icon + Terminal icon.
- **Contextual Log Out:**
  - Displays environment-aware label: `Log Out AG App`, `Log Out AG IDE`, or `Log Out AGY CLI`.

### 4. Smart Logout Commands (`src-tauri/src/agent_usage.rs`)
- `logout_antigravity`: Clears SQLite auth rows from `state.vscdb` (IDE domain).
- `logout_antigravity_cli`: Clears credential JSON files from `~/.gemini/` (Gemini Core domain, cleanly signing out both Desktop App and CLI).
